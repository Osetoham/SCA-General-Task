CONTAINER IMAGE IMMUTABILITY

An immutable image is an image that contains everything it needs to run the application, so obviously including your source code Docker containers are in fact immutable. This means that a running container never changes because in case you need to update it, the best practice is to create a new container with the updated version of your application and delete the old one.

  • A Docker image is analogous to a compiled program executable.
  • A Docker container is an instance of the docker image, and it's therefore one process that you started by invoking that program executable.

A given running process is immutable, in the sense that you don’t expect to modify the code for its program while it’s running. You would instead modify the source code, build a new executable, and start a new process to replace the current process.
